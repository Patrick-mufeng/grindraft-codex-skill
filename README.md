## 核心理念

磨稿的核心是一个 **"校准预测循环"**：

```
选题 → 写作 → 发布前盲预测 → 发布 → T+3天回收数据 → 复盘验证预测 → 优化评分体系 → 下一个选题更准
```

每一篇文章都会产生数据，数据会校正评分 rubric，rubric 让下一次预测更精准。这是一个**自我进化的系统**，越用越准。

### 三个核心阶段

| 阶段 | 包含 Skill | 目标 |
|------|-----------|------|
| **创作阶段** | init → trends → seed → write → humanize → illustrate → cover → polish → format | 从 0 到 1 完成一篇公众号文章 |
| **校准阶段** | predict → publish → retro → persona → bump | 发布后验证预测，驱动系统迭代 |
| **辅助工具** | recommend → status | 日常写作辅助 |

## 节点详解

### 1. 入口路由 — grindraft

**触发**：说"磨稿"或"grindraft"

**工作方式**：
- 调用 `scripts/phase.mjs` 检测当前项目阶段
- 根据阶段将用户路由到合适的 skill

**阶段检测逻辑**：

| 检测条件 | 阶段 | 行为 |
|---------|------|------|
| 没有 `.grindraft-state.json` | `init` | → 引导运行 grindraft-init |
| 没有选题候选 | `cold-start` | → 建议抓热点 → 聊选题 → 写文章 |
| 复盘文章 < 5 篇 | `calibrating` | → 按用户意图路由到对应 skill |
| 复盘文章 ≥ 5 篇 | `steady` | → 推荐复盘/升级/推荐选题 |

**特点**：不做任何业务逻辑，只做阶段检测和派发。

---

### 2. 初始化 — grindraft-init

**触发**：`磨稿初始化`

**用途**：第一次使用时的项目脚手架搭建。

**工作流程**：
1. 创建 `.grindraft-state.json`（全局状态文件）
2. 创建 `rubric_notes.md`（评分标准笔记）
3. 创建 `style_guide.md`（风格指南，随使用自动沉淀）
4. 创建 `WORKFLOW.md`、`STATUS.md`
5. 创建 `candidates.md`（选题池）、`articles/`（文章目录）

**两种模式**：

| 模式 | 适用场景 | 行为 |
|------|---------|------|
| `cold-start` | 全新开始 | 等权重 v0 rubric，前 5 篇进入校准期 |
| `calibration` | 已有历史文章 | 导入历史文章提取风格基线 + 初始权重拟合 |

---

### 3. 抓热点 — grindraft-trends

**触发**：`抓热点`

**用途**：从互联网拉取 AI 领域今日热点资讯，作为选题素材。

**工作流程**：
1. 调用 aihot API（`https://aihot.virxact.com`，公开接口，无需鉴权）
2. 对结果去重、粗分类（AI 模型 / AI 产品 / 行业动态 / 论文 / 技巧）
3. 写入 `candidates.md` 选题池

**输出示例**：
```
## 候选选题

- [2025-01-15] **Claude 4 发布** | 来源：aihot | 类型：AI 模型
  - 热度：🔥🔥🔥🔥
  - 角度建议：对普通用户意味着什么？
```

**扩展性**：可在 state 中配置更多数据源适配器（见 `adapters/trend-sources/`）。

---

### 4. 聊选题 — grindraft-seed

**触发**：`聊选题`

**用途**：通过深度对话挖掘你的个人经历和观察，收敛出一个**独特的写作角度**。

**为什么需要这个步骤**？
- AI 直接写的文章通篇正确的废话
- 你的真实经历 + 独特观点 = 读者真正想看的内容
- 这个 skill 的作用是**把你肚子里的货挖出来**

**工作流程**：
1. 读取 `candidates.md` 选题池 + `style_guide.md` 风格指南
2. 通过对话引导你回忆：最近遇到什么？观察到了什么？和别人的观点有什么不同？
3. 收敛到 1-3 个具体角度，供你选择
4. （可选）快速写一段角度验证草稿

---

### 5. 写文章 — grindraft-write

**触发**：`写文章`

**用途**：基于确定的角度，AI 生成公众号初稿。

**三级自主度**：

| 级别 | 名称 | 行为 | 适用场景 |
|------|------|------|---------|
| 🟢 Low | 低自主 | 写 50%，剩余留 TODO 标记让你补充 | 你有很多个人素材要填入 |
| 🟡 Medium | 中自主 | 写完整篇，标注 3 个可编辑点 | 大部分时候用这个 |
| 🔴 High | 高自主 | 写完整篇，直接可发布级 | 纯热点快讯或你完全信任 AI |

**输出结构**（`articles/{title}_{date}/draft.md`）：
```markdown
---
title: 文章的标题
date: 2025-01-15
status: draft
summary: 文章摘要
---

## 标题候选
1. 标题 A
2. 标题 B
3. 标题 C

## 正文
...
```

### 6. 去 AI 味 — grindraft-humanize

**触发**：`去 AI 味`

**用途**：4 层自检，逐层去除 AI 痕迹，让文章读起来像"人写的"。

**四层检查**：

| 层级 | 名称 | 检查内容 | 示例 |
|------|------|---------|------|
| **L1** | 硬规则 | 删除开头/结尾套话 | "在这个飞速发展的时代…"、"总的来说…" |
| **L2** | 语气清理 | 删除填充词、AI 惯用句式 | "值得注意的是"、"不可否认"、"从某种意义上说" |
| **L3** | 节奏扫描 | 检查段落长度、句式变化、长短句交替 | 连续 3 段超过 5 行 → 拆分 |
| **L4** | 人性终审 | 读起来像不像人写的？有没有"人味"？ | 加入口语、个人经历、情绪变化 |

**额外能力**：自动对比你的编辑和 AI 初稿，提取你的修改模式，沉淀到 `style_guide.md`。**只有出现 ≥2 次的模式才会成为正式规则**。

### 7. 配图 — grindraft-illustrate

**触发**：`配图`

**用途**：为文章生成内插图，使用"小黑（Xiaohei）"手绘风格。

**工作流程**：
1. 分析文章结构，识别可配图段落
2. 生成分镜列表（shot list）
3. 为每个分镜生成 16:9 的插图
4. 插入到文章对应位置

**风格参考**：
- 小黑风格：天马行空的手绘风
- 引用文件：`shared-references/xiaohei-ip.md`、`shared-references/composition-patterns.md`

---

### 8. 设计封面 — grindraft-cover

**触发**：`设计封面`

**用途**：基于文章的**情绪 × 领域 × IP**三维分析，从 40 套模板推荐封面方案。

**工作流程**：
1. **情绪分析**：文章基调是冷的？暖的？激昂的？轻松的？严肃的？
2. **领域匹配**：文章属于哪个领域（AI/职场/教育/生活…）？
3. **IP 匹配**：是否需要加入公众号 IP 元素？
4. 推荐 3 套封面模板
5. 生成预览 HTML（2.35:1 + 1:1 两种比例）+ 可选 PNG

**模板分类**：

| 维度 | 分类 |
|------|------|
| 情绪 | 冷 / 暖 / 激昂 / 轻快 / 严肃 |
| 领域 | AI 科技 / 职场成长 / 教育 / 生活感悟 / 商业… |

---

### 9. 打磨标题 — grindraft-polish

**触发**：`打磨标题`

**用途**：独立生成标题候选 + 封面 prompt（不启动预测流程时使用）。

**注意**：这个 skill **不会**写入预测，只是一个纯创作辅助工具。正式的盲预测必须在 `grindraft-predict` 中完成。

**输出**：
- 10-15 个标题候选，按钩力排序
- 封面配图 AI prompt

---

### 10. 排版 — grindraft-format

**触发**：`排版`

**用途**：将 Markdown 转换为公众号兼容的 HTML。

**工作流程**：
1. AI 根据文章主题设计一套配色主题（每次不同）
2. 将 Markdown 转为符合公众号规范的 HTML
3. 输出两份：

| 文件 | 用途 |
|------|------|
| `formatted.html` | 干净 HTML，直接复制粘贴到公众号编辑器 |
| `preview.html` | 手机模拟预览，查看实际效果 |

**特性**：
- 严格遵循公众号 CSS 白名单（`adapters/format/公众号HTML排版设计规范.md`）
- 安全 CSS：`color`、`font-size`、`background`、`margin`、`padding`、`border-radius` 等
- 危险 CSS：`font-family`（会被覆盖）、`line-height`（部分支持）、`letter-spacing`（不生效）
- 每篇文章配色不同，不重复

### 11. 启动预测 — grindraft-predict

**触发**：`启动预测`

**用途**：发布前的**盲预测**——这是整个校准体系的核心步骤。

**⚠️ 铁律：预测一旦写入，不可修改。**

**预测包含 7 个组成部分**：

| # | 组件 | 内容 |
|---|------|------|
| 1 | **7 维评分** | 对标题钩力、叙事牵引力等 7 个维度逐一打分（1-5） |
| 2 | **桶级赌注** | 预测文章落在哪个表现等级（S/A/B/C/D/E） |
| 3 | **概率分布** | 对各桶级的信心分布，如"B 级 60%，C 级 30%，A 级 10%" |
| 4 | **锚点对比** | 与哪篇历史文章类比，预期表现如何 |
| 5 | **反事实推演** | 如果预测错了，最可能错在哪里？ |
| 6 | **校准假设** | 当前校准状态，预测信心等级 |
| 7 | **Channel B** | 调用 `grindraft-score-blind` 隔离子 agent 独立打分 |

**冷启动简化**（前 5 篇）：只需 7 维评分 + 桶级赌注 + "无锚点"标记。

---

### 12. 已发布 — grindraft-publish

**触发**：`已发布`

**用途**：登记文章的发布信息。

**工作流程**：
1. 输入发布的文章 URL
2. 系统记录发布时间戳
3. 更新状态文件：标记文章为"已发布"
4. 添加到待复盘列表（T+3 天回收数据）

**约束**：**绝不修改**预测部分的内容。

---

### 13. 复盘 — grindraft-retro

**触发**：`复盘`

**用途**：T+3 天后回收实际数据，验证预测，提取观察。

**工作流程**：
1. **T+3 到达**：系统提示可以复盘了
2. **数据输入**：用户从公众号后台粘贴实际数据（阅读量、分享量、收藏量、留言量等）
3. **假设验证**：逐一对比预测的 7 维分数 vs 实际表现
4. **衍生指标计算**：打开率、完读率、分享率、收藏率、留言率
5. **观察提取**：从偏差中提取可复用的写作洞察
6. **跨样本检测**：新观察与历史数据是否一致？是 → 写入 rubric；否 → 标记为待验证

**数据来源说明**：由于公众号没有公开 API，数据由用户手动粘贴。这是系统的**唯一数据入口**。

---

### 14. 画像推导 — grindraft-persona

**触发**：`推导画像`

**用途**：从文章的留言数据推导读者群体画像。

**画像维度**：
- 职业分布推测
- 阅读动机（求知 / 娱乐 / 追热点 / 解决具体问题）
- 评论质量分布
- 关注点偏好

**与 rubric 的关系**：
- Rubric 回答"如何评分"
- Persona 回答"谁在读"

---

### 15. Rubric 升级 — grindraft-bump

**触发**：`升级`

**用途**：当校准数据积累到一定程度后，对评分体系进行升级。

**触发条件**：
- 新复盘数据与现有 rubric 的排名出现系统性偏差
- 有 ≥3 个跨样本一致的观察等待纳入

**升级流程**：
1. **全量重打分**：用新权重/新维度对**所有历史文章**重新打分
2. **排名一致性检验**：新旧排名必须 ≥80% 一致（即 4/5 的文章排名不变）
3. **跨模型审计**：可选调用其他 AI 模型独立审计评分
4. **通过则更新**：写入新 rubric，git 记录旧版

**版本历史**：
- **v0**（冷启动）：等权重（所有维度 ×1.0）
- **v1**（≥5 篇校准后）：加权权重（如 HK×2.0, NR×2.0, QA×1.5…）

---

### 16. 选题推荐 — grindraft-recommend

**触发**：`推荐选题`

**用途**：对 `candidates.md` 选题池中的候选进行综合排序。

**排序因子**：
- 热度趋势
- 与风格指南的匹配度
- 历史同类文章表现
- 时效性（是否即将过时）
- 你的写作疲劳度（避免连续写同一类话题）

### 18. 盲打分引擎 — grindraft-score-blind

**触发**：由 `grindraft-predict` 内部调用，不对外暴露。

**用途**：作为 Channel B 的隔离打分 agent。

**设计原理**：
- 在独立隔离的子 agent 中运行
- **看不到**文章原文、state 文件、任何历史预测
- 只接收打分所需的样本片段
- 保证打分是真正的"盲打"，不受已有信息干扰

## 七维评分体系

这是整个校准系统的核心。每一篇文章在发布前都按这 7 个维度打分，然后与实际数据对比，持续优化。

| 编号 | 缩写 | 维度 | 权重 | 衡量什么 | 对应的公众号信号 |
|------|------|------|------|---------|----------------|
| 1 | **HK** | 标题钩力 | ×2.0 | 标题让人想点吗？ | 打开率 / 阅读量 |
| 2 | **NR** | 叙事牵引力 | ×2.0 | 读得下去吗？ | 完读率 |
| 3 | **QA** | 金句锚点 | ×1.5 | 有让人截图发朋友圈的句子吗？ | 划线点击 / 在看 |
| 4 | **UT** | 实用密度 | ×1.5 | 有多少"收藏了以后用"的内容？ | 收藏率 |
| 5 | **DT** | 思辨激发度 | ×1.0 | 让人想留言讨论吗？ | 留言量 & 留言质量 |
| 6 | **EP** | 情绪峰值 | ×1.0 | 有"哇塞"或泪点时刻吗？ | 分享率 |
| 7 | **SC** | 结构闭环 | ×0.5 | 结构完整不烂尾吗？ | 软影响（留言质量） |

**综合得分公式**：
```
composite = (HK×2.0 + NR×2.0 + QA×1.5 + UT×1.5 + DT×1.0 + EP×1.0 + SC×0.5) / 9.5 × 2.0
```

**等级映射**：

| 等级 | 综合分范围 | 含义 |
|------|-----------|------|
| S | 9.0 - 10.0 | 爆款潜力 |
| A | 7.5 - 8.9 | 优质 |
| B | 6.0 - 7.4 | 良好 |
| C | 4.5 - 5.9 | 及格 |
| D | 3.0 - 4.4 | 低于预期 |
| E | 0 - 2.9 | 不及格 |

## 三大铁律

### 1. 盲预测不可修改

预测一旦写入就锁定。如果你确实预测错了需要重来，只能新写一个 `_redo.md`，原预测文件保留不动。git 会记录所有版本。

### 2. Rubric 升级必须全量重打分

每次升级 rubric（改权重、增删维度），必须对所有历史校准样本重新打分，且新旧排名一致性 ≥80%（4/5）才算通过。

### 3. Rubric 是工作台，不是博物馆

被数据反复推翻的观察要删除，而不是留着做陈列。想找回？git 历史就是你的档案馆。

## 发布节奏策略

系统内置了三个阶段的发布节奏建议（见 `shared-references/cadence-protocol.md`）：

| 阶段 | 频率 | 策略 |
|------|------|------|
| 🚀 启动期 | 每周 3-4 篇 | 快速积累校准数据，优先数量 |
| 📈 成长期 | 每周 2 篇 | "1 篇稳定 + 1 篇实验"，质量优先 |
| 🏆 成熟期 | 每周 1-2 篇 | 精准打击，每篇都是精品 |

**四池管理**：选题池 / 草稿池 / 终稿池 / 待复盘池，确保创作节奏不中断。
